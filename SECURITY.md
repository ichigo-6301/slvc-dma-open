# Security Policy

Before relying on this policy, repository maintainers enable GitHub Private
Vulnerability Reporting for `ichigo-6301/slvc-dma-open`. Report potential
security issues through the repository security advisory channel, not through a
public issue.

Reports must use public-safe reproductions. Do not include credentials, PDKs,
licenses, proprietary integration details, generated IP, or private tool logs.
The project has no response-time commitment; maintainers will assess whether an
issue affects the released public profile and coordinate a provenance-preserving
fix when appropriate.
